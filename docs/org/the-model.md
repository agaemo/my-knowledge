# The Model（SaaS 組織モデル）

SaaS ビジネスにおける収益組織の分業モデル。Salesforce が実践し、Jacco van der Kooij の著書 *The Model*（2018年）で体系化された。カスタマーサクセス・インサイドセールスが標準化された背景にある考え方。

## なぜ存在するか

従来の営業モデルは「1人の営業がリード獲得からクロージング・アフターフォローまで全部担う」形だった。SaaS ではこの構造が機能しにくい。

```
従来型（パッケージ売り）:
  売ったら終わり → 1回の大きな売上

SaaS の収益構造:
  売っても毎月の継続課金 → 解約されると全部失う
  → 「売る」と「使い続けてもらう」を同等に重視する必要がある
```

専門性の高い分業にすることで、各フェーズの質を上げながらスケールできる。

## ファネルと役割

```
[マーケティング]
  リードを生成・育成する
       ↓
[SDR / インサイドセールス]
  リードを精査・アポ獲得
       ↓
[AE（Account Executive）]
  商談・クロージング
       ↓
[CSM（Customer Success Manager）]
  オンボーディング・活用支援・リテンション・拡販
```

### Marketing（マーケティング）

認知・リード獲得・ナーチャリングを担う。MQL（Marketing Qualified Lead）をセールスに渡す役割。

### SDR（Sales Development Representative）

インバウンドで来たリードを精査し、商談アポを設定する。**売ることが仕事ではなく、適切な見込み客かを判断することが仕事**。

- **SDR** — インバウンドリード（問い合わせ・トライアル登録など）を対応
- **BDR**（Business Development Representative） — アウトバウンドで新規開拓。特にエンタープライズ向け

### AE（Account Executive）

商談を受け取り、提案・デモ・交渉・クロージングまでを担う。新規顧客の獲得が主なミッション。

### CSM（Customer Success Manager）

契約後の顧客が**成功体験を得られるよう**支援する。解約防止（リテンション）と追加受注（アップセル・クロスセル）が評価指標になる。

「サポート」との違い：サポートは問題が起きたとき対応する。CSM は問題が起きる前に能動的に関与し、顧客が目標を達成できるよう動く。

### SE（Solutions Engineer）/ プリセールスエンジニア

技術的な商談を支援する役割。AE と同行してデモ・POC・技術的な質問対応を担う。FDE（Forward Deployed Engineer）に近いが、プリセールス（契約前）寄り。

## 健全性を測る指標

### NRR（Net Revenue Retention / 売上維持率）

既存顧客からの収益が、解約・縮小・拡大を経て前期比でどう変化したかを示す指標。

```
NRR = (期末の既存顧客収益) ÷ (期初の既存顧客収益) × 100

  NRR > 100% → アップセルが解約を上回っている（良い状態）
  NRR = 100% → 解約とアップセルがゼロサム
  NRR < 100% → 解約が進んでいる
```

SaaS 企業の健全性を示す最重要指標のひとつ。新規顧客を獲得し続けなくても NRR が 120% なら収益は自然に成長する。

### チャーンレート（Churn Rate）

一定期間に解約した顧客・収益の割合。NRR の逆の視点。

## PLG（Product-Led Growth）

The Model（Sales-Led）の対概念。プロダクト自体が獲得・活性化・拡大のエンジンになるモデル。The Model と PLG は対立ではなく、PLG で個人・SMB を獲得しエンタープライズは The Model で攻めるハイブリッド戦略が多い。

詳細は [成長戦略](/business/growth-strategy) を参照。

## エンジニアとの接点

The Model を理解することで、自分たちが作るプロダクトがどの文脈で使われるかがわかる。

| 機能・領域 | 関係するロール |
|---|---|
| オンボーディングフロー・チュートリアル | CSM（顧客の初期成功を支援） |
| 管理者向けダッシュボード・利用状況分析 | CSM（顧客のヘルスチェック）|
| CRM 連携（Salesforce など） | AE・SDR の業務フロー |
| プロダクト内通知・ヒント | PLG の活性化施策 |
| API・インテグレーション | SE・FDE が技術的な商談で使う |
