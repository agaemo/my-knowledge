# 単一障害点（SPOF）

## 何か

Single Point of Failure。システム内で1つ壊れると全体が停止するコンポーネント。

冗長化されていない箇所はすべてSPOFになりうる。

## なぜ問題か

そのコンポーネントの可用性が、システム全体の可用性の上限になる。
他の部分をどれだけ信頼性高く作っても、SPOFがあれば意味がない。

例：99.99%のサービスに99.9%の外部依存が1つあれば、全体のSLAは99.9%を超えられない。

## 典型的な発生箇所

- **データベース**：プライマリが1台のみ（レプリカなし）
- **ロードバランサー**：1台構成
- **外部サービス**：フォールバックなしで依存
- **共有キャッシュ**：全リクエストが通過する単一のRedisノード
- **デプロイパイプライン**：CIが1リージョンのみ

## 対策

| アプローチ | 内容 |
|---|---|
| 冗長化 | 複数インスタンスを並列稼働させ、フェイルオーバーを自動化 |
| ステートレス化 | 状態を持たせないことで横スケールを可能にする |
| フォールバック | 依存先が落ちても代替手段で動作を継続 |
| 非同期化 | 依存先の応答を待たないアーキテクチャにする |

## 見つけ方

依存関係グラフを描き、「この1点が落ちたら何が止まるか」を問い続ける。
インシデント後のポストモーテムで発見されることも多い。
