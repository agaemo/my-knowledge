# SRE（Site Reliability Engineering）

システムの信頼性・可用性を設計・維持するための概念とパターン。

## 信頼性の指標

- [SLI / SLO / SLA](./slo-sli-sla) — 信頼性を定量化してエラーバジェットで管理する

## 障害のパターン

- [単一障害点（SPOF）](./single-point-of-failure) — 1点の故障がシステム全体を止める構造
- [カスケード障害](./cascade-failure) — 局所的な障害が連鎖的に全体に波及する現象
- [リトライストーム](./retry-storm) — 一斉リトライが回復中のサービスを再び落とす現象

## 障害への対策パターン

- [サーキットブレーカー](./circuit-breaker) — 障害のある依存先への呼び出しを遮断するパターン
- [バルクヘッド](./bulkhead) — コンポーネント間でリソースを隔離し障害の波及を防ぐ
- [バックプレッシャー](./backpressure) — 処理能力を超えた入力を上流に伝えてレートを抑制する

## 障害からの学習

- [ポストモーテム](./postmortem) — インシデント後に原因・再発防止策を記録・共有するプロセス
