# SRE（Site Reliability Engineering）

システムの信頼性・可用性を設計・維持するための概念とパターン。

- [SREとは何か](./what-is-sre) — 誕生の経緯・DevOpsとの違い・トイル・自動化の段階
- [SLI / SLO / SLA](./slo-sli-sla) — 信頼性を定量化してエラーバジェットで管理する
- [エラーバジェット](./error-budget) — 許容できる障害量を定量化し、リリースと信頼性のトレードオフを管理する
- [障害パターン](./failure/) — 単一障害点・カスケード障害・リトライストームなど、何が・なぜ起きるか
- [対策パターン](./resilience/) — タイムアウト・サーキットブレーカー・バルクヘッドなど、どう防ぐか
- [ポストモーテム](./postmortem) — インシデント後に原因・再発防止策を記録・共有するプロセス
