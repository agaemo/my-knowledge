# RAID（冗長構成）

複数の物理ディスクを組み合わせて1つの論理ディスクとして扱い、**冗長性・可用性・パフォーマンス**のいずれか（またはその組み合わせ）を向上させるストレージ技術。

## なぜ存在するか

単一のディスクは物理的な故障（ヘッドクラッシュ・モーター故障・経年劣化）によりいつか必ず読めなくなる。RAIDはディスクを単一障害点にしないための仕組みで、1台が故障してもデータを失わず・サービスを止めずに済む構成を実現する。

## 主なRAIDレベル

### RAID 0（ストライピング）

```
[データA][データB][データC][データD]
    ↓          ↓
[ディスク1]  [ディスク2]
 A, C         B, D
```

- 複数ディスクにデータを分散して書き込む（ストライピング）
- 読み書き速度が向上する
- **冗長性なし**。1台でも故障するとすべてのデータを失う
- 用途：一時ファイル・キャッシュなど、失っても問題ないデータの高速I/O

### RAID 1（ミラーリング）

```
[データ]
   ↓  ↓（同じデータを2台に書く）
[ディスク1]  [ディスク2]
```

- 2台に同一のデータを書き込む（ミラーリング）
- 1台が故障しても残りから読み続けられる
- 容量効率は50%（2台で1台分）
- 用途：OSディスク・小規模サーバーの起動ドライブ

### RAID 5（ストライピング＋パリティ）

```
[ディスク1]  [ディスク2]  [ディスク3]
  A データ    B データ     A+B のパリティ
  C+D のパリティ  C データ  D データ
```

- データとパリティ（誤り訂正情報）を複数ディスクに分散
- 1台の故障なら残りのディスクとパリティからデータを復元できる
- 容量効率が高い（3台なら2/3が使える）
- 2台以上の同時故障には耐えられない
- 用途：ファイルサーバー・NASなど読み込みが多いワークロード

### RAID 6（ダブルパリティ）

- RAID 5 のパリティを2つに増やしたもの
- **2台同時故障**まで耐えられる
- 容量効率はRAID 5よりやや低下
- 用途：大容量HDDを使う環境（容量が大きいほど再構築に時間がかかり、その間の2台目故障リスクが高い）

### RAID 10（1+0）

- RAID 1（ミラーリング）をRAID 0（ストライピング）で束ねる
- 冗長性と速度を両立できるが、容量効率は50%
- 用途：データベースサーバーなど読み書き性能と可用性を同時に求める場面

## RAIDはバックアップではない

RAIDは**物理的なディスク故障**への対策であり、以下には効果がない。

- ファイルの誤削除・上書き（ミラーリングされてすぐ反映される）
- ランサムウェアによる暗号化
- 火災・水害などの物理的な災害
- ストレージコントローラーの故障

別拠点へのバックアップや、スナップショット（ポイントインタイムリカバリ）と組み合わせて使う。
