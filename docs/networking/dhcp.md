# DHCP（Dynamic Host Configuration Protocol）

ネットワークに接続したデバイスへIPアドレス・サブネットマスク・デフォルトゲートウェイ・DNSサーバーなどの設定を自動的に配布するプロトコル。

## なぜ存在するか

ネットワーク上のデバイスが増えると、すべての端末にIPアドレスを手動で割り当てるのは現実的でない。DHCPはデバイスが接続するだけで必要な設定を自動取得できるようにすることで、管理コストを大幅に削減する。

## 動作の流れ（DORA）

```
1. Discover  クライアント → ブロードキャスト
             「DHCPサーバーはいますか？」

2. Offer     DHCPサーバー → クライアント
             「このIPアドレスを使いませんか？（リース期間付き）」

3. Request   クライアント → ブロードキャスト
             「そのIPアドレスをください」

4. Acknowledge  DHCPサーバー → クライアント
             「割り当てました。設定情報をどうぞ」
```

Discoverは自分のIPが未確定のためブロードキャストで送信する。

## 配布される情報

| 設定項目 | 説明 |
|---------|------|
| IPアドレス | クライアントに割り当てるアドレス |
| サブネットマスク | ネットワーク範囲の定義 |
| デフォルトゲートウェイ | 外部ネットワークへの出口 |
| DNSサーバーアドレス | 名前解決に使うリゾルバーのIP |
| リース期間 | IPアドレスの有効期限 |

## リース期間と更新

DHCPはIPアドレスを永続的に割り当てるのではなく、**リース**（貸し出し）として期限付きで付与する。クライアントはリース期間の50%が経過した時点で更新要求を出し、期限切れを防ぐ。デバイスが切断すると、そのIPは別のデバイスに再利用される。

## ペネトレーションテストでの意味

### DHCP スターベーション（枯渇攻撃）

偽のクライアントとして大量のDiscover/Requestを送り、DHCPサーバーが配布できるIPアドレスのプールを使い果たす。新規に接続するデバイスがIPを取得できなくなる（DoS）。

### DHCP スプーフィング（偽DHCPサーバー）

スターベーションで正規のDHCPサーバーを無効化した後、攻撃者が偽のDHCPサーバーを立てる。デフォルトゲートウェイやDNSサーバーを攻撃者のIPに向けることで、通信を傍受する（中間者攻撃）。

```
正規の構成:
  クライアント → 正規DHCP → ゲートウェイ → インターネット

DHCP スプーフィング後:
  クライアント → 偽DHCP → 攻撃者マシン（傍受）→ インターネット
```

### 防御

- **DHCPスヌーピング**: スイッチが信頼できるポートからのDHCP応答のみを通過させる
- **Dynamic ARPインスペクション**: 偽のARP応答も合わせてブロックする
