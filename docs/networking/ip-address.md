# IPアドレス

## なぜ存在するか

インターネット上のデバイスを一意に識別するための番号。手紙の住所に相当し、データを正しい宛先に届けるために使われる（OSI参照モデルのL3・ネットワーク層）。

## IPv4

32ビットのアドレス。`192.168.0.1` のように8ビットずつ `.` で区切った10進数4つで表記する。

- アドレス空間：約43億（2³²）
- 例：`192.168.1.1`、`8.8.8.8`（Google DNS）

**枯渇問題**
インターネットの普及でIPv4アドレスが足りなくなった。NATでプライベートIPを共有することで延命しているが、根本解決はIPv6への移行。

### 特殊なアドレス

| アドレス | 用途 |
|---------|------|
| `127.0.0.1` | ループバック（自分自身）。`localhost` の実体 |
| `0.0.0.0` | 全インターフェースを表す（サーバーのListen設定など） |
| `169.254.x.x` | リンクローカル（DHCP失敗時に自動割当） |
| `255.255.255.255` | ブロードキャスト（同一ネットワーク全体への送信） |
| `10.x.x.x` `172.16-31.x.x` `192.168.x.x` | プライベートアドレス（→ [NAT](./nat.md)参照） |

### ループバック（127.0.0.0/8）

`127.0.0.1` は「自分自身」を指す特殊アドレス。実際には `127.0.0.0/8`（127.0.0.1〜127.255.255.254）の範囲全体がループバック用に予約されている。この範囲に送ったパケットはNICに到達せず、OSのネットワークスタック内で折り返す。

`localhost` はホスト名で、`/etc/hosts` で `127.0.0.1`（またはIPv6の `::1`）に解決される。OSの設定によっては `::1` が優先される場合があり、サーバーがIPv4でしか待ち受けていないと接続できないことがある。

```bash
# /etc/hosts の典型的な記述
127.0.0.1   localhost
::1         localhost
```

**使われる場面**
- ローカル開発サーバー（`http://localhost:3000`）
- 同一マシン内のプロセス間通信（NginxからApp Serverへの転送など）
- データベースのローカル接続

### 0.0.0.0（全インターフェース）

サーバーが特定のインターフェースではなく**全てのインターフェース**で待ち受けることを表す。

1台のマシンには複数のネットワークインターフェース（NIC）があることが多い（ループバック用の `lo`、有線の `eth0`、無線の `wlan0` など）。

```
マシン
├── lo:    127.0.0.1     ← ループバック
├── eth0:  192.168.1.10  ← LAN
└── eth1:  203.0.113.1   ← WAN（グローバルIP）
```

| バインドアドレス | 外部からのアクセス | ループバックからのアクセス |
|---|---|---|
| `127.0.0.1:80` | 不可 | 可 |
| `192.168.1.10:80` | LAN内のみ可 | 不可 |
| `0.0.0.0:80` | 全インターフェースで可 | 可 |

開発中に `localhost` 以外からアクセスさせたい場合（スマートフォンから確認するなど）は `0.0.0.0` でバインドする必要がある。

### 169.254.x.x（リンクローカル / APIPA）

DHCP サーバーからIPアドレスを取得できなかった場合に、OSが自動的に割り当てるアドレス範囲（169.254.0.0/16）。同一リンク（同じL2ネットワーク）内のみで有効で、ルーティングされない。

このアドレスが見えた場合はほぼ「DHCPが失敗している」サインと考えてよい。

AWS EC2 では `169.254.169.254` がインスタンスメタデータサービス（IMDS）のエンドポイントとして使われる。この場合はDHCP失敗ではなく、AWSが意図的に設定した特別な用途。

## IPv6

128ビットのアドレス。16ビットずつ `:` で区切った16進数8つで表記する。

- アドレス空間：約340澗（2¹²⁸）≒ 340兆の1兆倍の1兆倍
- 例：`2001:0db8:85a3:0000:0000:8a2e:0370:7334`

**省略表記のルール**
- 各グループ先頭の `0` は省略可：`0db8` → `db8`
- 連続する `0000` のグループは `::` で一度だけ省略可：`2001:db8::1`

### 特殊なアドレス

| アドレス | 用途 |
|---------|------|
| `::1` | ループバック（IPv4の`127.0.0.1`に相当） |
| `fe80::/10` | リンクローカルアドレス（同一ネットワーク内のみ有効） |
| `::ffff:0:0/96` | IPv4マップドアドレス（IPv6環境でIPv4アドレスを表現） |

## IPv4 と IPv6 の比較

| | IPv4 | IPv6 |
|-|------|------|
| ビット長 | 32ビット | 128ビット |
| 表記 | `192.168.0.1` | `2001:db8::1` |
| アドレス数 | 約43億 | 事実上無限 |
| ヘッダー | 可変長（オプションあり） | 固定長（シンプル） |
| NAT | 必要（アドレス不足のため） | 不要（全デバイスにグローバルIP） |
| セキュリティ | IPsecはオプション | IPsecを標準サポート |
| 現状 | 主流だが枯渇 | 普及拡大中 |

## デュアルスタック

IPv4とIPv6を同時にサポートする構成。現在の移行期における標準的なアプローチ。サーバーは両方のアドレスでListenし、クライアントはIPv6を優先して接続する（Happy Eyeballs）。

## 実務での登場シーン

- **`0.0.0.0` でサーバーをListen**：全インターフェースで待ち受ける設定。`localhost`（127.0.0.1）だとループバックのみで外部から届かない
- **DNSのAレコードとAAAAレコード**：A = IPv4、AAAA = IPv6のIPアドレスを返すレコード
- **セキュリティグループ・ファイアウォール**：`0.0.0.0/0` は全IPv4、`::/0` は全IPv6を意味するCIDR
- **ログのIPアドレス**：クライアントIPがIPv6形式（`::ffff:192.168.1.1`）で出ることがある
