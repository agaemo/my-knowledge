# IPアドレス

## なぜ存在するか

インターネット上のデバイスを一意に識別するための番号。手紙の住所に相当し、データを正しい宛先に届けるために使われる（OSI参照モデルのL3・ネットワーク層）。

## IPv4

32ビットのアドレス。`192.168.0.1` のように8ビットずつ `.` で区切った10進数4つで表記する。

- アドレス空間：約43億（2³²）
- 例：`192.168.1.1`、`8.8.8.8`（Google DNS）

**枯渇問題**
インターネットの普及でIPv4アドレスが足りなくなった。NATでプライベートIPを共有することで延命しているが、根本解決はIPv6への移行。

### 特殊なアドレス

| アドレス | 用途 |
|---------|------|
| `127.0.0.1` | ループバック（自分自身）。`localhost` の実体 |
| `0.0.0.0` | すべてのネットワークインターフェースを表す（サーバーのListen設定など） |
| `255.255.255.255` | ブロードキャスト（同一ネットワーク全体への送信） |
| `10.x.x.x` `172.16-31.x.x` `192.168.x.x` | プライベートアドレス（→ [NAT](./nat.md)参照） |

## IPv6

128ビットのアドレス。16ビットずつ `:` で区切った16進数8つで表記する。

- アドレス空間：約340澗（2¹²⁸）≒ 340兆の1兆倍の1兆倍
- 例：`2001:0db8:85a3:0000:0000:8a2e:0370:7334`

**省略表記のルール**
- 各グループ先頭の `0` は省略可：`0db8` → `db8`
- 連続する `0000` のグループは `::` で一度だけ省略可：`2001:db8::1`

### 特殊なアドレス

| アドレス | 用途 |
|---------|------|
| `::1` | ループバック（IPv4の`127.0.0.1`に相当） |
| `fe80::/10` | リンクローカルアドレス（同一ネットワーク内のみ有効） |
| `::ffff:0:0/96` | IPv4マップドアドレス（IPv6環境でIPv4アドレスを表現） |

## IPv4 と IPv6 の比較

| | IPv4 | IPv6 |
|-|------|------|
| ビット長 | 32ビット | 128ビット |
| 表記 | `192.168.0.1` | `2001:db8::1` |
| アドレス数 | 約43億 | 事実上無限 |
| ヘッダー | 可変長（オプションあり） | 固定長（シンプル） |
| NAT | 必要（アドレス不足のため） | 不要（全デバイスにグローバルIP） |
| セキュリティ | IPsecはオプション | IPsecを標準サポート |
| 現状 | 主流だが枯渇 | 普及拡大中 |

## デュアルスタック

IPv4とIPv6を同時にサポートする構成。現在の移行期における標準的なアプローチ。サーバーは両方のアドレスでListenし、クライアントはIPv6を優先して接続する（Happy Eyeballs）。

## 実務での登場シーン

- **`0.0.0.0` でサーバーをListen**：全インターフェースで待ち受ける設定。`localhost`（127.0.0.1）だとループバックのみで外部から届かない
- **DNSのAレコードとAAAAレコード**：A = IPv4、AAAA = IPv6のIPアドレスを返すレコード
- **セキュリティグループ・ファイアウォール**：`0.0.0.0/0` は全IPv4、`::/0` は全IPv6を意味するCIDR
- **ログのIPアドレス**：クライアントIPがIPv6形式（`::ffff:192.168.1.1`）で出ることがある
