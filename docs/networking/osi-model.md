# OSI参照モデル / TCP/IPモデル

## なぜ存在するか

ネットワーク通信を「役割ごとの層」に分割することで、各層を独立して設計・実装・交換できるようにするための概念モデル。「L4ロードバランサ」「L7スイッチ」など、ネットワーク製品や設定の文脈で頻繁に登場する。

## OSI参照モデル（7層）

| 層番号 | 名前 | 役割 | 代表例 |
|--------|------|------|--------|
| L7 | アプリケーション層 | ユーザー向けプロトコル | HTTP, HTTPS, DNS, FTP |
| L6 | プレゼンテーション層 | データ形式変換・暗号化 | TLS/SSL, JSON, XML |
| L5 | セッション層 | セッション管理・同期 | RPC, NetBIOS |
| L4 | トランスポート層 | エンドツーエンドの通信制御 | TCP, UDP |
| L3 | ネットワーク層 | IPアドレスによるルーティング | IP, ICMP |
| L2 | データリンク層 | MACアドレスによる同一ネットワーク内転送 | Ethernet, Wi-Fi |
| L1 | 物理層 | 電気信号・光信号の伝送 | ケーブル, 光ファイバー |

## TCP/IPモデル（4層）

実際のインターネットプロトコルスタックはTCP/IPモデルで設計されている。OSIは理論モデル、TCP/IPが実装モデルと理解するとよい。

| TCP/IP層 | 対応するOSI層 |
|----------|--------------|
| アプリケーション層 | L5〜L7 |
| トランスポート層 | L4 |
| インターネット層 | L3 |
| ネットワークアクセス層 | L1〜L2 |

## いつ使うか（実務での登場シーン）

**L4 vs L7 ロードバランサの選定**
- L4（トランスポート層）：TCPレベルで転送。HTTPの中身は見ない。低レイテンシ・高スループット向き
- L7（アプリケーション層）：HTTPヘッダー・パスを見てルーティング。パスベースルーティング・SSL終端・認証に使う

**ファイアウォール設定**
- L3：IPアドレスで制御
- L4：ポート番号で制御（`80`, `443` を許可するなど）
- L7：アプリケーションコンテンツで制御（WAFなど）

**障害調査**
- pingが通る → L3まで正常
- TCPコネクションが張れる → L4まで正常
- HTTPレスポンスが返らない → L7の問題

## TCPとUDPの違い

| | TCP | UDP |
|-|-----|-----|
| 接続 | コネクション型（3ウェイハンドシェイク） | コネクションレス |
| 信頼性 | 再送制御あり、順序保証あり | なし |
| 速度 | 低速 | 高速 |
| 用途 | HTTP, SMTP, DB接続 | DNS, 動画ストリーミング, ゲーム, QUIC |
