# VPC / サブネット / CIDR

## なぜ存在するか

クラウド上のリソースをそのままインターネットに晒すと危険。VPCは「仮想的に隔離されたプライベートネットワーク」を作り、外部から直接アクセスできないリソース（DB・内部APIなど）を保護する。

## VPC（Virtual Private Cloud）

クラウド内に作る論理的なプライベートネットワーク。AWS・GCP・Azureすべてに同概念がある。VPC内のリソースは原則、VPC外からアクセスできない。

## CIDR（Classless Inter-Domain Routing）

IPアドレスの範囲を表記する記法。`192.168.0.0/24` のように「ベースIP / プレフィックス長」で表す。

**プレフィックス長の意味**
- `/24` → 先頭24ビットがネットワーク部 → 残り8ビットがホスト部 → 2⁸ = 256アドレス
- `/16` → 65,536アドレス
- `/32` → 1アドレス（特定のIP1つを指す）

| CIDR | アドレス数 | 用途 |
|------|-----------|------|
| `/8` | 約1,677万 | 大規模VPC |
| `/16` | 65,536 | 標準的なVPC |
| `/24` | 256 | サブネット単位でよく使う |
| `/28` | 16 | 小さいサブネット |
| `/32` | 1 | 特定ホストの指定（セキュリティグループなど） |

## サブネット（Subnet）

VPCをさらに小さく分割したネットワーク。**パブリックサブネット**と**プライベートサブネット**に分けるのが基本設計。

```
VPC: 10.0.0.0/16
├── パブリックサブネット: 10.0.1.0/24  ← インターネットゲートウェイ接続
│   └── ALB, NATゲートウェイ, Bastionホスト
└── プライベートサブネット: 10.0.2.0/24  ← 外部からアクセス不可
    └── アプリサーバー, RDS, ElastiCache
```

**パブリックサブネット**：インターネットゲートウェイ（IGW）へのルートがある。外部からの通信を受け付ける。  
**プライベートサブネット**：IGWへのルートがない。外部からは届かない。インターネットへの発信はNATゲートウェイ経由。

## セキュリティグループ vs NACL

| | セキュリティグループ | NACL（ネットワークACL） |
|-|---------------------|----------------------|
| 適用対象 | インスタンス（EC2など） | サブネット |
| ステート | ステートフル（返りの通信を自動許可） | ステートレス（入出力を個別に設定） |
| ルール | 許可のみ | 許可・拒否の両方 |
| 優先度 | なし（全ルールを評価） | 番号順に評価、最初にマッチしたルールが適用 |

## 可用性ゾーン（AZ）をまたいだ構成

障害に備え、サブネットを複数のAZに分散させる。

```
VPC
├── AZ-a: パブリック (10.0.1.0/24) + プライベート (10.0.2.0/24)
└── AZ-b: パブリック (10.0.3.0/24) + プライベート (10.0.4.0/24)
```

ALBは複数AZのサブネットを指定することで、AZ障害時にも自動的に正常なAZへトラフィックを流す。
