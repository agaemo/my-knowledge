# NAT / ポートフォワーディング

## なぜ存在するか

IPv4アドレスは約43億個しかなく、全デバイスにグローバルIPを割り当てることは不可能。NATはプライベートIPアドレスを持つ複数のデバイスが1つのグローバルIPを共有してインターネットに出ていく仕組み。

## プライベートIPアドレスとは

インターネットに直接ルーティングされない、LAN内だけで使えるIPアドレス帯。

| 範囲 | CIDR | 用途 |
|------|------|------|
| 10.0.0.0 〜 10.255.255.255 | `10.0.0.0/8` | 大規模LAN・クラウドVPC |
| 172.16.0.0 〜 172.31.255.255 | `172.16.0.0/12` | 中規模LAN |
| 192.168.0.0 〜 192.168.255.255 | `192.168.0.0/16` | 家庭・小規模オフィス |

## NAT（Network Address Translation）の仕組み

```
デバイス（192.168.1.10:5000）
  → ルーター（NAT変換）
    → インターネット（グローバルIP: 203.0.113.1:50000）
      → 外部サーバー（93.184.216.34:443）
```

ルーターは「192.168.1.10:5000 → 203.0.113.1:50000」のマッピングテーブルを保持し、戻りのパケットを正しいデバイスに届ける。ポート番号も使って変換するため NAPT（Network Address Port Translation）または PAT（Port Address Translation）とも呼ばれる。

## クラウドにおけるNATゲートウェイ

プライベートサブネット内のリソース（アプリサーバー・Lambdaなど）がインターネットに発信したいとき（パッケージ取得・外部API呼び出しなど）に使う。

```
プライベートサブネット → NATゲートウェイ（パブリックサブネット） → IGW → インターネット
```

NATゲートウェイは受信方向のインバウンドを遮断するため、外部からプライベートサブネットには届かない。

## ポートフォワーディング

NAT機器のグローバルIPの特定ポートへの通信を、プライベートIP:ポートに転送するルール。

```
外部 → ルーター（203.0.113.1:8080）
      ↓ ポートフォワーディング設定
     → 内部PC（192.168.1.10:80）
```

**用途**
- ゲームサーバーをローカルPCで動かして外部に公開
- 自宅サーバーへのSSH接続（外部22番ポート → 内部の特定PC）
- 開発中のWebサーバーを外部に一時公開

クラウドでは セキュリティグループ・ロードバランサ・Ingressで同様のことを実現するため、ポートフォワーディングを意識する場面は少ない。ローカル開発環境（ngrok・SSHポートフォワード）での文脈でよく登場する。

## SSHポートフォワーディング

ローカルの特定ポートをSSHトンネル越しにリモートのポートに転送する。

```bash
# ローカルの5432番 → リモートサーバー経由でDB(10.0.0.5:5432)に接続
ssh -L 5432:10.0.0.5:5432 user@bastion-server
```

プライベートサブネット内のRDSに踏み台（Bastionホスト）経由でローカルのDBクライアントから接続するときによく使う。
