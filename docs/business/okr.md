# OKR（Objectives and Key Results）

目標（Objective）とその達成度を測る主要な成果指標（Key Results）を組み合わせた目標管理フレームワーク。  
Intel の Andy Grove が 1970年代に開発し、1999年に John Doerr が Google に持ち込んで普及した。現在は Google・LinkedIn・Twitter・Uber・Airbnb・Spotify など多くのテック企業が採用している。

## 構造

```
Objective（目標）
  何を達成したいか。定性的・インスピレーショナルな表現。
  「〜になる」「〜を実現する」

  Key Result 1（主要な成果指標）
  Key Result 2
  Key Result 3
    各 KR は定量的・測定可能。達成/未達成が明確にわかる。
```

**例：**

```
Objective: ユーザーが毎日使いたくなるプロダクトにする

  Key Result 1: DAU / MAU 比率を 30% → 45% に改善する
  Key Result 2: 7日間リテンション率を 40% → 55% に改善する
  Key Result 3: NPS を 32 → 45 に改善する
```

## なぜ存在するか

多くの組織が陥る問題：
- 「何に集中すべきか」が人によってバラバラ
- タスクをこなすことが目的化し、成果につながっているか不明
- 目標が曖昧で達成したかどうか判断できない

OKR はこれを解決するために「**大胆な目標（Objective）**」と「**測定可能な成果（Key Results）**」を明示的に分ける。数値がなければ Key Result とは言わない。

## KPI との違い

| | OKR | KPI |
|---|---|---|
| 目的 | 変革・成長・達成したい姿 | 日常業務の健全性モニタリング |
| 視点 | 「何を目指すか」 | 「うまく動いているか」 |
| 更新頻度 | 四半期・年次 | 常時モニタリング |
| 達成目標 | 60〜70% 達成が理想（野心的な設定） | 100% 達成が基本 |

OKR と KPI は対立ではなく補完関係。KPI でベースラインを維持しながら、OKR で次のステージを目指す。

## 「60〜70% 達成が理想」の理由

OKR は**達成が難しい**目標を設定する（Moonshot / Stretch Goal）。100% 達成できるなら目標が低すぎる。Google では 70% 達成を「成功」と定義し、100% 達成した場合は目標設定を見直す文化がある。

これにより「安全な目標」への引き下げを防ぎ、組織全体が高い基準を目指し続ける。

## 運用のポイント

**透明性：** OKR は全員が見られるようにする。CEO から現場エンジニアまで、誰が何を目指しているかが分かる状態。

**四半期サイクル：** 多くの企業が四半期（Q1〜Q4）単位で設定・振り返りを行う。

**ボトムアップとトップダウンの組み合わせ：** 会社・部門・チーム・個人の各レベルで OKR を設定し、上位の目標とアライメントさせる。ただし、上から割り振るだけでなく個人が自分で設定する部分も残す。

**OKR と評価を切り離す：** 達成率を直接評価・給与に連動させると、安全な目標しか設定されなくなる。Google は OKR を参考情報にとどめている。
