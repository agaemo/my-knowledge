# SWOT分析

自社・事業を「強み（Strengths）」「弱み（Weaknesses）」「機会（Opportunities）」「脅威（Threats）」の4象限に整理し、戦略の方向性を導くフレームワーク。内部要因（強み・弱み）と外部要因（機会・脅威）、ポジティブな要因とネガティブな要因を掛け合わせて分類する。

## 4象限

```
            内部要因          外部要因
         ┌─────────────┬─────────────┐
ポジティブ │  Strengths   │ Opportunities │
         │    強み      │     機会      │
         ├─────────────┼─────────────┤
ネガティブ │  Weaknesses  │    Threats    │
         │    弱み      │     脅威      │
         └─────────────┴─────────────┘
```

- **Strengths（強み）**: 競合に対して優位な内部資源・能力（技術力・ブランド・特許など）
- **Weaknesses（弱み）**: 競合に対して劣る内部要因（資金力・人材不足・知名度など）
- **Opportunities（機会）**: 活用できる外部環境の変化（市場拡大・規制緩和・競合の撤退など）
- **Threats（脅威）**: 対処すべき外部環境の変化（新規参入・規制強化・代替品の登場など）

## なぜ存在するか

「自社の強み・弱みは何か」と「市場の機会・脅威は何か」は別々に考えると見落としが多い。SWOT分析は内部要因と外部要因を1枚に並べることで、強みだけ見て市場の脅威を見落とす・機会だけ見て自社の弱みで取りこぼすことに気づかない、といった片寄った戦略判断を防ぐ。

## クロスSWOT（TOWS分析）への展開

4象限を並べただけでは「で、何をすべきか」が出てこない。実務では象限同士を掛け合わせて打ち手を導く**クロスSWOT分析**まで行うのが一般的。

| | 機会（O） | 脅威（T） |
|---|---|---|
| **強み（S）** | 積極化戦略：強みで機会を獲りに行く | 差別化戦略：強みで脅威の影響を防ぐ |
| **弱み（W）** | 弱点強化戦略：弱みを補い機会を取り逃さない | 防衛・撤退戦略：最悪のシナリオを避けるため縮小・撤退も検討する |

## よくある失敗

**ブレストで終わる：** 4象限に項目を書き出すだけで終わり、クロスSWOTで打ち手まで落とし込まない。

**社内目線だけで埋める：** 「強み」を経営陣の願望で埋めてしまい、顧客や競合との比較（[3C分析](/business/strategy-analysis/3c-analysis)）の裏付けがない。

**項目が多すぎる：** 各象限に10個以上書き出すと優先順位がつかない。重要な3〜5個に絞る。

## 3Cとの関係

SWOTの「機会・脅威」は市場・競合の分析が前提になる。[3C分析](/business/strategy-analysis/3c-analysis)で市場（Customer）・競合（Competitor）を分析した結果を、SWOTの外部要因（機会・脅威）の材料として使う、という組み合わせ方が一般的。

## いつ使うか

- 事業戦略・新規事業の方向性を議論する初期フェーズ
- 経営陣・チームで現状認識を揃えたいとき
- 強み・弱みだけでなく外部環境も含めて網羅的に洗い出したいとき

現状の整理には向くが、時間とともに状況がどう変化するかは捉えられない。競争環境の力学まで見たい場合は[ワードリーマップ](/business/strategy-analysis/wardley-map)を併用する。
