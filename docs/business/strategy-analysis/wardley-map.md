# ワードリーマップ（Wardley Map）

Simon Wardley が考案した、事業やシステムを構成する要素を「価値連鎖（誰が何を必要としているか）」と「進化の度合い（その要素がどれだけコモディティ化しているか）」の2軸で地図化する戦略分析手法。

## なぜ存在するか

[SWOT分析](/business/strategy-analysis/swot)や[3C分析](/business/strategy-analysis/3c-analysis)は現状を一覧化するのに向いているが、「次に何をすべきか」「競合はどう動くか」という時間軸・力学までは捉えにくい。

ワードリーマップは、構成要素が時間とともに進化する（未知の発明 → 独自開発 → 製品化 → コモディティ化）という前提を地図に組み込むことで、

- どこに投資すべきか（まだコモディティ化していない差別化要素）
- どこを外部に任せるべきか（コモディティ化した要素を自前で持つ無駄）
- 競合・市場がどう動くか（進化の力学から予測する）

を視覚的に議論できるようにする。

## 2つの軸

```
縦軸: 価値連鎖（Value Chain）
  ユーザーに近い「見える価値」を上に、
  それを支える「見えない基盤」を下に配置する

横軸: 進化（Evolution）
  Genesis（未知の発明） → Custom-Built（独自開発） → Product/Rental（製品化） → Commodity/Utility（コモディティ化）
```

```
例: オンラインショップの場合

顧客の購買体験 ●───────────────────────────●
                │
   決済機能      │              ●（Stripe等のSaaS）
                │
   サーバー      │                            ●（クラウド・コモディティ）
                │
   電力          │                                          ●（電気・完全にユーティリティ）
        Genesis ──────────────────────────────────────── Commodity
```

要素は左（未成熟・差別化要因）から右（コモディティ・誰でも使える）に向かって移動していく。技術が成熟・標準化していく一方向の動きで、逆方向には基本的に動かない。

## 進化の4段階

| 段階 | 特徴 | 例 |
|---|---|---|
| **Genesis**（未知の発明） | 不確実・実験的・差別化の源泉 | 新しいML手法の研究 |
| **Custom-Built**（独自開発） | 動くが個別最適・属人化しやすい | 自社専用の認証システム |
| **Product / Rental**（製品化） | ベストプラクティス化・選択肢が複数ある | SaaSの認証サービス |
| **Commodity / Utility**（コモディティ化） | 標準化・低コスト・差別化要因にならない | クラウドの仮想マシン・電力 |

## 使い方

```
1. ユーザー（誰が価値を必要としているか）を地図の頂点に置く
2. ユーザーが必要とする価値連鎖を、上から下へ依存関係として並べる
3. 各要素を進化の軸上に配置する
4. 進化の方向（左→右）を踏まえて、投資・外部化・撤退の判断を議論する
```

差別化要因（Genesis・Custom-Built側の要素）に投資を集中し、コモディティ化した要素はクラウド・SaaS・OSSなど外部の標準解に任せる、というのが典型的な打ち手。

## 3C・SWOTとの違い

| | 3C分析 / SWOT分析 | ワードリーマップ |
|---|---|---|
| 視点 | 静的な現状のスナップショット | 時間とともに進化する力学 |
| 出力 | 強み・弱み・機会・脅威の一覧 | 構成要素の配置図と移動の予測 |
| 向いている問い | 「今、何が強み・弱みか」 | 「次に何が起きるか、どこに投資すべきか」 |

3C・SWOTで現状を洗い出した後、その構成要素をワードリーマップで時間軸に展開する、という組み合わせ方もできる。

## いつ使うか

- build vs buy（自前開発するか、SaaS・OSSに任せるか）を判断したいとき
- 技術投資の優先順位を「差別化要因かコモディティか」で議論したいとき
- 競合・市場の構造変化（進化）を予測して先手を打ちたいとき

参加者がコモディティ化の概念を理解していないと議論が噛み合わないため、導入時はワークショップ形式で軸の意味から揃えるのが実務上の定石。
