# Vision・Mission・Value

組織の存在意義と行動規範を定義する3つの概念。経営の根幹として、意思決定・採用・プロダクト方針・文化のすべての基準になる。

## 3つの定義

**Vision**（ビジョン）  
会社が目指す将来の姿。「どこへ向かうか」。10〜20年先を見据えた野心的・インスピレーショナルな表現。

**Mission**（ミッション）  
会社が存在する理由。「なぜ存在するか」「誰のどんな問題を解くか」。Visionより具体的で、日々の意思決定の判断基準になる。

**Value**（バリュー）  
組織がどのように行動するかの原則。「どう行動するか」。採用・評価・文化の基準になる。

## 有名な例

| 企業 | Vision / Mission | Value の例 |
|---|---|---|
| Google | Mission: 「世界中の情報を整理し、世界中の人々がアクセスできて使えるようにする」 | Be a good citizen, Focus on the user |
| Amazon | Vision: 「地球上で最もお客様を大切にする会社」 | Customer Obsession, Bias for Action |
| Patagonia | Mission: 「私たちは、故郷である地球を救うためにビジネスを営む」 | — |
| メルカリ | Mission: 「あらゆる人の可能性を引き出すために、豊かさの循環を生み出す世界をつくる」 | — |

## なぜ存在するか

組織が小さいうちは、創業者の意図が自然に全員に伝わる。人が増えるにつれて「なぜこれをするのか」「どちらを選ぶべきか」という判断が個人に委ねられる。

VMV がなければ：
- プロダクトの方向性が部署ごとにバラバラになる
- 採用で「どんな人を採るか」の基準がない
- 社員が意思決定に迷うたびにエスカレーションが必要になる

VMV があれば：
- 「これはミッションに合っているか？」が意思決定の共通言語になる
- 採用基準・評価基準が Valueから導かれる
- 現場が自律的に判断できる

## 他フレームワークとの階層関係

```
Vision（10〜20年先の姿）
  ↓ 実現するために
Mission（存在理由・解く課題）
  ↓ 体現するために
Value（行動原則）
  ↓ 達成するために
North Star Metric（コアバリューを表す指標）
  ↓ 測定するために
OKR / KGI（期の目標）
  ↓ 監視するために
KPI（日常指標）
```

## Vision と Mission の混同について

日本では Vision と Mission が混同されることも多く、一方だけ定義している企業もある。厳密に両方定義する必要はなく、「なぜ存在するか・どこへ向かうか」が組織全体で共有されていることが本質。

## エンジニアとの接点

- **プロダクト判断**: 「この機能はミッションに合っているか」が優先順位判断の基準になる
- **採用**: エンジニアの採用面接でも Value との整合性が評価される
- **OKR**: Objective の出発点は Mission/Vision から来るべきで、技術的な施策を事業目標と接続するための橋渡しになる
