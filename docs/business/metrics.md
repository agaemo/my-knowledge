# スタートアップの重要指標

スタートアップの健全性・成長性を測るための指標群。投資家との会話・社内の意思決定・採用候補者向けの説明など、あらゆる場面で使われる。

## 成長ベンチマーク

### T2D3（Triple, Triple, Double, Double, Double）

SaaS スタートアップがユニコーン（時価総額 10億ドル）に到達するための成長軌跡の目安。  
Battery Ventures の Neeraj Agrawal が提唱。Salesforce・Workday・ServiceNow などの分析から導き出された。

```
ARR の成長倍率:
  Year 1 → Year 2: ×3（Triple）
  Year 2 → Year 3: ×3（Triple）
  Year 3 → Year 4: ×2（Double）
  Year 4 → Year 5: ×2（Double）
  Year 5 → Year 6: ×2（Double）

5年間で累計 72倍（3 × 3 × 2 × 2 × 2 = 72）
```

具体的な数値で見ると：

| 年 | ARR の例 |
|---|---|
| Year 1（基準）| $1M |
| Year 2 | $3M |
| Year 3 | $9M |
| Year 4 | $18M |
| Year 5 | $36M |
| Year 6 | $72M |

**投資家にとっての意味：** T2D3 の軌跡に乗っている SaaS は、大型 VC 投資の対象として評価されやすい。自社の成長ペースがこのベンチマークと比較してどこにいるかを把握することで、調達タイミングや事業目標の設定に使われる。

**あくまで目安：** T2D3 はあらゆる SaaS が目指すべき絶対基準ではなく、急成長型スタートアップの一つのリファレンス。市場規模・ビジネスモデル・ブートストラップ戦略によっては異なる成長曲線が正解になる。

## 売上に関する指標

### ARR / MRR

**MRR（Monthly Recurring Revenue）** — 月次経常収益。毎月繰り返し発生するサブスクリプション収益。  
**ARR（Annual Recurring Revenue）** — 年次経常収益。MRR × 12。

```
MRR の構成要素:
  New MRR    : 新規顧客からの収益
  Expansion  : 既存顧客のアップグレードによる増加
  Churned MRR: 解約・ダウングレードによる減少

Net New MRR = New MRR + Expansion MRR - Churned MRR
```

SaaS ビジネスでは一時的な売上より ARR・MRR で成長を測る。「ARR 1億円」が初期スタートアップの一つのマイルストーンとして語られることが多い。

### NRR（Net Revenue Retention）

既存顧客から得られる収益の維持率。[The Model](/org/the-model) ページも参照。

## コスト・存続に関する指標

### バーンレート（Burn Rate）

1ヶ月に使っている現金の純減額。

```
グロスバーン: 総支出（給与・オフィス・インフラなど）
ネットバーン: グロスバーン - 収益
            → 実際にキャッシュが減るスピード
```

### ランウェイ（Runway）

今の資金が尽きるまでの期間。

```
ランウェイ（月）= 手元現金 ÷ ネットバーンレート

例: 手元に 5,000万円、月 500万円 ネットバーン → ランウェイ 10ヶ月
```

次の資金調達が完了するまでのランウェイが 6〜12ヶ月を切ると警戒ラインとされる。調達には数ヶ月かかるため、余裕を持って動き始める必要がある。

## 顧客に関する指標

### CAC（Customer Acquisition Cost）

1顧客を獲得するためのコスト。

```
CAC = 営業・マーケティングの総コスト ÷ 獲得顧客数
```

### LTV（Life Time Value）

1顧客が契約期間中にもたらす収益の総額。

```
LTV = 平均 MRR ÷ 月次チャーンレート

例: 月 5万円の顧客が平均 24ヶ月契約 → LTV = 120万円
```

### LTV / CAC 比率

ビジネスの収益性を示す最重要指標の一つ。

```
LTV / CAC > 3 が健全とされる目安
  → 顧客1人を獲得するコストの3倍以上を回収できる

LTV / CAC < 1: 獲得コストを回収できていない（赤字成長）
```

CAC を下げるか LTV を上げるかが SaaS の根本的な課題。PLG はCAC を下げる戦略、CSM による活用支援は LTV を上げる戦略。

## ビジネス成立に関する概念

### PMF（Product-Market Fit）

**プロダクトが特定の市場のニーズにぴったり合っている状態。**  
Marc Andreessen が提唱した概念。「良いプロダクトを作れば売れる」のではなく、「その市場が本当に求めているか」が問われる。

```
PMF 前: どれだけ売っても顧客が定着しない・口コミが広がらない
PMF 後: 売り込まなくても顧客が増える・解約されにくい・紹介が自然に起きる
```

PMF の有無を測る簡易的な問い（Sean Ellis テスト）：  
「このプロダクトが使えなくなったらどう感じますか？」→ 「非常に残念」と答えるユーザーが 40% 以上いれば PMF に近い。

### ピボット（Pivot）

事業の方向性を大きく転換すること。PMF が見つからない場合に、ターゲット市場・課題・プロダクトの前提を変える判断。

Instagram はもともと位置情報ゲームアプリだった。Slack はゲーム会社の社内ツールだった。ピボットは失敗ではなく、学習に基づく戦略的な転換。

### ユニットエコノミクス（Unit Economics）

1顧客単位の収益構造。LTV > CAC であること（ユニットエコノミクスがポジティブ）がスケールの前提条件。
