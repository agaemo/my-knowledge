# コマンドインジェクション

ユーザー入力にOSコマンドを埋め込み、サーバー上で任意のシェルコマンドを実行させる攻撃。SQLインジェクションと同じ構造で、対象がDBではなくOSシェルになる。

## 仕組み

ユーザー入力をそのままシェルコマンドの引数に使うと、入力値がシェル構文として解釈される。

```
正常な処理: ping <ユーザー入力のホスト名>
攻撃入力:  example.com; cat /etc/passwd
実行されるコマンド: ping example.com; cat /etc/passwd
→ システムのパスワードファイルが出力される

攻撃入力:  example.com && rm -rf /var/www
→ Webサーバーのファイルが削除される
```

セミコロン（`;`）・パイプ（`|`）・AND（`&&`）・バックティック（`` ` ``）など、シェルのメタキャラクタが悪用される。

## 対策

- **シェルを経由しない**: `exec()` ではなくコマンドと引数を分けて渡す（例: Pythonの `subprocess.run(["ping", host], ...)` ）。シェルを介さなければメタキャラクタが解釈されない
- **ユーザー入力でコマンドを組み立てない**: コマンドはコード側で固定し、入力値はフラグやIDとして使う設計にする
- **入力バリデーション**: ホスト名なら英数字・ハイフン・ドットのみを許可するなど、許可する文字セットを絞る
- **最小権限の原則**: サーバープロセスの実行権限を最小化し、侵害されても被害を限定する
