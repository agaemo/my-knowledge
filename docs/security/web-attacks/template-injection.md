# テンプレートインジェクション

テンプレートエンジンの構文を悪意ある入力として送り込み、サーバーやブラウザ上で任意のコードを実行させる攻撃。

## SSTI（Server-Side Template Injection）

Jinja2・Twig・Freemarker などサーバー側のテンプレートエンジンが対象。ユーザー入力をテンプレート文字列として評価してしまう場合に発生する。

```
入力値: {{ 7*7 }}
レスポンス: 49  ← テンプレートとして評価されている証拠

入力値: {{ config.items() }}  （Jinja2の例）
→ サーバーの設定情報・秘密鍵などが漏洩する
```

## CSTI（Client-Side Template Injection）

AngularJS・Vue などフロントエンドフレームワークのテンプレート構文が対象。ユーザー入力がテンプレートとしてDOMに挿入されると発生する。

AngularJS ではテンプレート式にコンストラクタ経由の関数呼び出しを埋め込むことでスクリプト実行につながる。XSS の亜種として扱われることが多い。

## 確認方法

テンプレート式（例: 数式）を入力欄に入れて計算結果が表示されれば、テンプレートとして評価されている可能性がある。

## 対策

- **ユーザー入力をテンプレート文字列に渡さない**: テンプレートの構造はコード側で固定し、ユーザー入力は値として渡す
- **テンプレートエンジンのサンドボックス**: 一部のエンジンが提供するサンドボックスモードを使う（回避される場合もある）
- **出力エスケープ**: テンプレート構文として解釈される前に無害化する
