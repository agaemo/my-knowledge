# ディレクトリトラバーサル

ファイルパスに `../` などを含む入力を送り込み、公開を意図していないファイルにアクセスする攻撃。パストラバーサルとも呼ばれる。

## 仕組み

ユーザーから受け取ったファイル名をそのままパスに使うと、上位ディレクトリへの移動が可能になる。

```
リクエスト: /files?name=../../etc/passwd
サーバー処理: /var/www/files/../../etc/passwd → /etc/passwd
→ システムのパスワードファイルが読み取られる
```

Webサーバーのドキュメントルート外にあるファイル（設定ファイル・秘密鍵・ソースコードなど）が対象になる。

## 対策

- **パスの正規化と検証**: 受け取ったパスを正規化（`../` の解決）したうえで、許可されたベースディレクトリ内に収まっているか確認する
- **ファイル名のホワイトリスト**: 想定するファイル名のパターン以外を拒否する
- **ユーザー入力をパスに使わない**: ファイル名の代わりにIDを受け取り、サーバー側でパスにマッピングする設計にする
