# マジックバイト・ファイルシグネチャ

ファイルの先頭数バイトに埋め込まれた固定値で、ファイル種別を識別するために使われる。拡張子はOSやユーザーが自由に変更できるが、マジックバイトはファイルの内容そのものに含まれるため、より信頼性の高い種別判定として使われてきた。

## なぜ存在するか

拡張子はファイルシステム上の名前にすぎず、変更しても中身は変わらない。そのため画像ビューアやウイルススキャナ・セキュリティツールは、ファイルの中身を見て種別を判断する必要がある。マジックバイトはその判定基準として業界標準的に定着した仕組み。

## 主なマジックバイトの例

| ファイル種別 | 先頭バイト（16進数） | 表示 |
|------------|-------------------|------|
| JPEG | `FF D8 FF` | ÿØÿ |
| PNG | `89 50 4E 47 0D 0A 1A 0A` | ‰PNG.... |
| PDF | `25 50 44 46` | %PDF |
| ZIP | `50 4B 03 04` | PK.. |
| ELF（Linux実行ファイル） | `7F 45 4C 46` | .ELF |

## セキュリティにおける限界

マジックバイトはファイルの**先頭**しか見ない。そのため：

- **ポリグロットファイル**: JPEGヘッダー（`FF D8 FF`）の後にPHPコードを埋め込んだファイルは、画像として判定されつつPHPエンジンに渡すと実行される
- **バイト追加による偽装**: 正規の画像バイナリの末尾にスクリプトを連結することもできる

マジックバイトのチェックは「完全に正しいファイルである証明」にはならない。ファイルアップロードの検証では、画像なら画像ライブラリで再エンコードするなど、中身を処理して無害化する必要がある。
