# Testing

テスト設計・戦略・手法に関する知識。

- [TDD](/testing/tdd) — テストを先に書いて設計を導くアプローチ
- [テストダブル](/testing/test-doubles) — Stub・Mock・Spy・Fake・Dummy の違いと使い分け
- [レガシーコードへのテスト追加](/testing/legacy-code-testing) — Seam・Characterization Test・Golden Master Testing
- [契約テスト](/testing/contract-testing) — サービス間のインターフェース互換性を自動検証する手法（Pact）
- [テスト戦略](/testing/testing-strategy) — テストの種類・比率・優先度を決める考え方
- [ヘッドレスブラウザ](/testing/headless-browser) — Playwright・Puppeteer を使った E2E テスト・スクレイピング・PDF 生成
- [Page Object Model](/testing/page-object-model) — E2E テストのページ操作をオブジェクトに閉じ込めてセレクター依存を分離するパターン
- [ミューテーションテスト](/testing/mutation-testing) — コードに意図的なバグを埋め込んでテストの品質を評価する手法
- [プロパティベーステスト](/testing/property-based-testing) — 入力を自動生成して「任意の入力でも成り立つ性質」を検証する手法
- [負荷テスト・パフォーマンステスト](/testing/load-testing) — スループット・レイテンシ・限界値を測定してシステムの性能特性を明らかにする手法
- [スナップショットテスト](/testing/snapshot-testing) — 出力を保存して意図しない変化を自動検出する手法
- [ビジュアルリグレッションテスト](/testing/visual-regression-testing) — スクリーンショット比較で見た目の劣化を検出する手法
- [フレーキーテスト](/testing/flaky-test) — コードを変えていないのに結果が変わるテストの原因と対策
- [QA エンジニア](/testing/qa-engineer) — テスターとの違い・役割の実態・チームへの組み込み方
