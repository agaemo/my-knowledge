# メトリクス設計（RED / USE / Four Golden Signals）

「何を計測するか」を体系化したフレームワーク。  
計測できないものは改善できない。適切なメトリクスを選ぶことが、問題の早期発見と正確な診断につながる。

## なぜ体系化が必要か

メトリクスは無数に取れる。何でも計測すると：

- アラートが多すぎて重要なものが埋もれる（アラート疲労）
- ダッシュボードが複雑になり、障害時に何を見ればいいか分からなくなる
- 問題の原因が「何のメトリクスを見れば分かるか」が不明確になる

フレームワークは「何を見るべきか」の共通言語を提供する。

## RED メソッド

**リクエスト駆動のサービス**（API・マイクロサービス）に向いている。Tom Wilkie（Grafana Labs）が提唱。

| 文字 | 指標 | 意味 |
|---|---|---|
| **R** — Rate | リクエスト数/秒 | どれくらいのトラフィックが来ているか |
| **E** — Errors | エラー率 | リクエストのうち何%が失敗しているか |
| **D** — Duration | レイテンシ | リクエストの処理にかかる時間（P50・P95・P99） |

```
「今 API が遅い」を調査するとき:
  Rate  → トラフィックが急増していないか
  Error → エラーが増えていないか（エラーが速く処理されて見かけ上速くなることがある）
  Duration → P99 が上がっていないか（一部のリクエストだけ遅いか全体か）
```

## USE メソッド

**インフラ・リソース**（サーバー・DB・キュー）の監視に向いている。Brendan Gregg（Netflix）が提唱。

| 文字 | 指標 | 意味 |
|---|---|---|
| **U** — Utilization | 使用率 | リソースがどれくらい使われているか（CPU 70%など） |
| **S** — Saturation | 飽和度 | リソースの限界を超えて待機しているか（キューの長さなど） |
| **E** — Errors | エラー数 | リソースレベルのエラー数 |

```
「DB が重い」を調査するとき:
  Utilization → CPU・メモリ・ディスク IO の使用率
  Saturation  → コネクション待ち・クエリキューの長さ
  Errors      → デッドロック数・タイムアウト数
```

## Four Golden Signals

Google の SRE 本で提唱。大規模サービスで何を監視するかの基本として広く参照される。

| シグナル | 内容 |
|---|---|
| **Latency**（レイテンシ） | リクエストの処理時間。成功と失敗を分けて計測する（エラーが速くて正常が遅い場合を見逃さないため） |
| **Traffic**（トラフィック） | システムへの需要量（リクエスト数/秒・帯域など） |
| **Errors**（エラー） | 失敗するリクエストの割合。明示的なエラー（5xx）と暗黙的なエラー（誤ったデータを返す）を分けて考える |
| **Saturation**（飽和） | システムのリソースがどれくらい逼迫しているか。飽和に近づくと Latency が増加し始める |

RED の Rate ≈ Traffic、Duration ≈ Latency であり、概念は重なる。Four Golden Signals は SRE の観点でより広いシステムを対象にしている。

## パーセンタイルの重要性

レイテンシは平均値ではなくパーセンタイルで見る。

```
100リクエストのうち:
  P50（中央値）: 50ms → 半数のユーザーが体験する速さ
  P95: 500ms          → 上位5%が体験する遅さ
  P99: 2000ms         → 上位1%が体験する遅さ

平均: 70ms  ← P99 の問題を隠してしまう
```

**重要なユーザーほど P99 に引っかかりやすい**（ヘビーユーザーはリクエスト数が多い）。SLA の設定には P99 を使うことが多い。

## フレームワークの使い分け

| 監視対象 | 使うフレームワーク |
|---|---|
| Web API・マイクロサービス | RED |
| サーバー・DB・キューなどインフラ | USE |
| サービス全体の健全性（SRE 視点） | Four Golden Signals |

まず「今何が問題か」の診断に RED/USE を使い、SLO の設定や SRE ダッシュボード設計に Four Golden Signals を使うのが実践的なアプローチ。
