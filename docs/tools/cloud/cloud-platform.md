# クラウドプラットフォーム

インターネット経由でコンピューティングリソース（サーバー・ストレージ・ネットワーク・ソフトウェア）を提供するサービス形態。従来のオンプレミスに対する概念。

## オンプレミスとの違い

### オンプレミス
自社でサーバーを購入・設置・運用する形態。物理的な機器の調達から電源・冷却・ネットワーク・OSの管理まで全て自社が担う。

**特徴:**
- 初期投資が大きい（機器購入・データセンター費用）
- スケールアップに時間がかかる（機器調達が必要）
- セキュリティ・コンプライアンスを自社で完全制御できる
- 一定以上の規模では長期的にコストが低くなる場合がある

### クラウド
サービス事業者のインフラを使用量に応じて利用する形態。物理的なインフラ管理から解放される。

**特徴:**
- 初期投資なし・従量課金
- 数分でスケールアウト可能
- インフラ管理の手間が減り、アプリ開発に集中できる
- データの保管場所・管理はクラウド事業者に依存する

## サービスモデル

責任範囲によって3つのモデルに分かれる。

```
                    SaaS
              PaaS
        IaaS
オンプレ

↑ ユーザーが管理する範囲が広い
↓ クラウドが管理する範囲が広い（楽になる）
```

### IaaS（Infrastructure as a Service）
仮想サーバー・ストレージ・ネットワークを提供。OSより上の層はユーザーが管理する。

- **例**: AWS EC2, Google Compute Engine, Azure Virtual Machines
- **向いている場面**: 既存のオンプレ構成をそのままクラウドに移行したい、細かいOS設定が必要

### PaaS（Platform as a Service）
アプリを動かすための実行環境を提供。OS・ミドルウェアの管理は不要。

- **例**: AWS Elastic Beanstalk, Google App Engine, Heroku
- **向いている場面**: インフラを意識せずアプリのデプロイに集中したい

### SaaS（Software as a Service）
完成したアプリをサービスとして提供。ユーザーはソフトウェアを使うだけ。

- **例**: Gmail, Slack, Salesforce, GitHub
- **向いている場面**: 自社で構築・運用する必要がないもの

## 主要プラットフォームの特徴

| プラットフォーム | 強み |
|----------------|------|
| AWS | サービス数が最多・シェアが最大・エコシステムが成熟 |
| GCP | データ分析・ML・AI。Kubernetes の生みの親 |
| Azure | Microsoft製品との親和性。エンタープライズ・官公庁での採用が多い |

## デプロイモデル

### パブリッククラウド
AWS・GCP・Azureのような、複数の企業が共有するクラウド環境。

### プライベートクラウド
自社専用のクラウド環境。オンプレミスに構築するか、クラウド事業者の専用環境を借りる。コンプライアンス要件が厳しい業種で使われる。

### ハイブリッドクラウド
オンプレミスとパブリッククラウドを組み合わせる。機密データはオンプレに置き、スケールが必要な処理はクラウドを使うといった使い分けができる。

## いつクラウドを選ぶか

- スタートアップ・小〜中規模: 初期コストを抑えたい → **パブリッククラウド**
- 負荷変動が大きい: スケールの柔軟性が必要 → **パブリッククラウド**
- 規制業種（金融・医療）: データ管理の制御が必要 → **プライベートまたはハイブリッド**
- 大規模かつ安定負荷: 長期的なコスト最適化 → **オンプレミスとの比較検討**
