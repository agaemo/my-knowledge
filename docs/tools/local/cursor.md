# Cursor

VSCodeをベースにAI操作を前提として再設計されたコードエディタ。コード補完・チャット・エージェントモードをエディタに深く統合しており、AIとの協調開発に特化している。

## なぜ存在するか

GitHub CopilotはVSCodeの「補助」として後付けされたが、CursorはAIが開発フローの中心にあることを前提に設計された。コードベース全体を把握した上での補完・修正・生成ができる点がCopilotとの大きな違い。

## VSCodeとの関係

VSCodeのフォーク。拡張機能・設定・キーバインドはほぼそのまま移行できる。VSCode向けの拡張機能も使える。

## 主要機能

### Tab補完
現在のコンテキストを読んで次の編集を予測し、Tabで確定できる。単語補完ではなく「次にやりたいこと」を予測する。

### Chat（⌘ + L）
エディタ右側にチャットパネルが開く。開いているファイル・選択範囲・コードベース全体を参照しながら質問・相談ができる。

```
@ファイル名    # 特定ファイルを参照させる
@コードベース  # プロジェクト全体を参照させる
@ドキュメント  # ドキュメントを参照させる
```

### Composer（⌘ + I）
複数ファイルにまたがる変更を一括で生成・適用するエージェントモード。「この機能を追加して」という指示から、関連ファイルの編集・新規作成までを自動で行う。

### Rules
プロジェクトやユーザーごとにAIへの指示（コーディング規約・技術スタック・禁止事項）を`.cursor/rules/`に定義できる。

```markdown
<!-- .cursor/rules/general.mdc -->
- TypeScriptを使う
- コメントは日本語で書く
- any型は禁止
```

## よく使うショートカット（Mac）

| 操作 | ショートカット |
|------|--------------|
| Chatを開く | `⌘ + L` |
| Composerを開く | `⌘ + I` |
| インラインEdit | `⌘ + K` |
| Tab補完を確定 | `Tab` |
| Tab補完をスキップ | `Escape` |

## モデル選択

Claude・GPT-4o・Geminiなど複数のLLMを切り替えて使える。タスクによって得意なモデルを使い分けられる。

## VSCodeとの使い分け

| 状況 | 向いているエディタ |
|------|------------------|
| AIをフル活用して実装を進めたい | Cursor |
| 軽量に素早くファイルを開きたい | VSCode |
| 拡張機能の互換性を最優先にしたい | VSCode |
| コードベース全体を把握した提案が欲しい | Cursor |
