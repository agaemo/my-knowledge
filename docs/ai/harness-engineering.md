# Harness Engineering

## なぜ存在するか

AIコーディングツールは素の状態では「プロジェクトのルール・アーキテクチャ・禁止事項」を知らない。毎回プロンプトで説明するのは非効率で、抜け漏れも起きる。

Harness Engineering は**AIが動く環境そのものを設計・構成する**こと。プロジェクト固有のコンテキスト・権限・自動化を組み込んだ環境を作ることで、AIの出力品質と一貫性を高める。

```
素のAI: 毎回「このプロジェクトでは〜」と説明が必要
Harness あり: 環境が知識・制約・安全策を保持している
```

現在は Claude Code・GitHub Copilot・Cursor など各ツールが独自の仕組みを持つが、**「AIへの常時コンテキスト」「権限制御」「自動化フック」「再利用コマンド」という構造はスタンダードになりつつある**。

## 構成要素

### 1. Instructions ファイル（常時コンテキスト）

AIが毎回読む「プロジェクトの憲法」。設計思想・規約・禁止事項を書く。

| ツール | ファイル |
|---|---|
| Claude Code | `CLAUDE.md` |
| GitHub Copilot | `.github/copilot-instructions.md` |
| Cursor | `.cursorrules` |
| Windsurf | `.windsurfrules` |

**書くべき内容**:
- プロジェクト概要・技術スタック
- アーキテクチャ上の決定と制約
- コーディング規約・命名ルール
- テスト方針
- 禁止事項（`.env` のコミット禁止など）

**書かなくて良い内容**: コードパターン・構造（コードから読み取れる）、git 履歴（git log で確認できる）

### 2. 権限制御（Safety Boundary）

AIが自律実行できる操作の範囲を明示的に定義する。「許可リスト」と「禁止リスト」で管理する。

```json
// Claude Code: .claude/settings.json
{
  "permissions": {
    "allow": [
      "Bash(npm run test:*)",
      "Bash(npm run build)",
      "Bash(git status)",
      "Read(**/*)",
      "Edit(**/*)"
    ],
    "deny": [
      "Bash(git push*)",
      "Bash(rm -rf*)"
    ]
  }
}
```

自動でやってよいこと・確認が必要なこと・禁止することを明確にすることで、意図しない操作を防ぐ。

### 3. 自動化フック

AIのアクション前後に自動で実行されるコマンド。AIが行動した直後にフィードバックを返し、問題を即座に検知できる。

```json
// Claude Code: PostToolUse フック
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "npm run typecheck 2>&1 | head -20"
      }]
    }]
  }
}
```

| タイミング | 用途 |
|---|---|
| ファイル編集後 | 型チェック・Lint の自動実行 |
| ツール実行前 | 危険操作のガード |
| セッション終了時 | 通知・ログ記録 |

### 4. 再利用コマンド・プロンプト

繰り返し使う指示をファイルとして定義し、呼び出せるようにする。

```markdown
<!-- Claude Code: .claude/commands/review.md  →  /review で呼び出し -->
以下の観点でコードレビューをしてください：
1. バグ・ロジックエラー
2. セキュリティ問題
3. アーキテクチャ規約への違反
4. テストの不足
```

GitHub Copilot では `.github/` 以下のカスタム指示、Cursor では Composer のルールとして同様の仕組みが提供される。

## 設計の考え方

### 一度書いて何度も使える仕組みにする

「毎回プロンプトで指示する」→「Instructions ファイルや再利用コマンドに組み込む」に変える。コンテキストの再説明コストをゼロにするのが目標。

### フィードバックループを短くする

フックでAIの行動直後に型チェック・テストを走らせると、AIが自分のエラーを見て即座に修正できる。人間のレビューを待たずにループが回る。

### グローバルとプロジェクトを分ける

| スコープ | 内容 |
|---|---|
| グローバル（個人設定） | 全プロジェクト共通のルール・好みのスタイル |
| プロジェクト | そのプロジェクト固有のアーキテクチャ・規約・権限 |

個人の好みはグローバルに、プロジェクト固有のルールはリポジトリに入れてチームで共有する。
