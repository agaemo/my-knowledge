# マルチモーダル AI

## なぜ存在するか

初期の LLM はテキストのみを扱うユニモーダルなモデルだった。しかし現実の情報はテキスト・画像・音声・動画など複数のモダリティに渡る。
マルチモーダル AI は複数のモダリティを統合的に処理することで、テキストだけでは表現・理解できない情報を扱えるようにする。

```
ユニモーダル: テキスト → モデル → テキスト
マルチモーダル: テキスト + 画像 + 音声 → モデル → テキスト・画像・音声
```

## モダリティの種類

| モダリティ | 入力例 | 出力例 |
|---|---|---|
| テキスト | 質問・指示 | 回答・要約・コード |
| 画像 | スクリーンショット・写真 | 説明・OCR・分析 |
| 音声 | 会話・録音 | 文字起こし・翻訳 |
| 動画 | 映像ファイル | 要約・シーン説明 |
| ドキュメント | PDF・スライド | 要約・Q&A |

## 従来の専用モデルとの違い

マルチモーダル以前は、モダリティごとに専用モデルを組み合わせていた。

```
従来の構成:
画像 → OCRモデル → テキスト → LLM → 回答

マルチモーダルLLM:
画像 + テキスト → LLM → 回答（文脈を統合して理解）
```

専用モデルのパイプラインでは「画像の文脈をテキストに変換する過程で情報が失われる」問題があった。マルチモーダル LLM は画像とテキストを同時に処理するため、視覚的な文脈を保ったまま推論できる。

## 代表的なユースケース

### 画像理解・分析

```
- UI のスクリーンショットから問題を診断する
- 図・グラフから数値や傾向を読み取る
- 手書きのメモやホワイトボードをテキスト化する
- 医療画像や設計図の説明を生成する
```

### ドキュメント処理

PDF・スライドをそのまま入力し、レイアウトや図表を含めて理解した上で Q&A や要約ができる。テキスト抽出だけでは失われる表・図の意味を保持できる。

### コード生成（ビジュアル → コード）

デザインのスクリーンショットや UI モックを渡して、対応するコードを生成させる。

### 音声インタラクション

音声入力 → 理解 → 音声出力をエンドツーエンドで処理する。音声認識（STT）と音声合成（TTS）を別々に組み合わせるより、文脈や感情を保持したまま処理できる。

## 主なモデル

| モデル | 提供元 | 対応モダリティ |
|---|---|---|
| GPT-4o | OpenAI | テキスト・画像・音声 |
| Claude 3.5 / 3 シリーズ | Anthropic | テキスト・画像・PDF |
| Gemini 1.5 / 2.0 | Google | テキスト・画像・音声・動画 |
| LLaVA / LLaMA 3.2 | Meta（OSS） | テキスト・画像 |

## 実装例（画像 + テキストの入力）

```ts
import Anthropic from '@anthropic-ai/sdk'
import * as fs from 'fs'

const client = new Anthropic()

/** 画像ファイルをBase64エンコードして送信する */
async function analyzeImage(imagePath: string, question: string) {
  const imageData = fs.readFileSync(imagePath)
  const base64Image = imageData.toString('base64')

  const response = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 1024,
    messages: [
      {
        role: 'user',
        content: [
          {
            type: 'image',
            source: {
              type: 'base64',
              media_type: 'image/png',
              data: base64Image,
            },
          },
          {
            type: 'text',
            text: question,
          },
        ],
      },
    ],
  })

  return response.content[0].type === 'text' ? response.content[0].text : ''
}
```

## 注意点

### トークンコスト

::: warning 画像はトークンを大量消費する
テキストと比べて大幅にコストが上がる。用途に合わせて解像度をリサイズする。
- 低解像度（512×512 程度）→ 数百トークン
- 高解像度（2048×2048）→ 数千トークン
:::

### 幻覚（ハルシネーション）

::: warning 画像の細部は誤認しやすい
小さな数字・似た文字を誤認することがある。重要な情報は人間が確認するか、信頼スコアを設ける。
:::

### プライバシー

::: danger 個人情報を含む画像の外部送信に注意
個人の顔・個人情報が含まれる画像を外部 API に送信する場合は、利用規約とデータ保持ポリシーを必ず確認する。
:::

## いつ使うか

- テキストだけでは表現できない情報（図・画像・レイアウト）を処理したい
- 専用モデルのパイプラインを減らして構成をシンプルにしたい
- ユーザーが画像や音声で入力できるインタフェースを作りたい
