# OCR（光学文字認識）

## なぜ存在するか

紙の書類・画像・PDFなど、デジタルテキストとして存在しない情報はコンピュータが直接処理できない。
OCR（Optical Character Recognition）は画像の中の文字を認識してテキストデータに変換し、検索・編集・データ処理を可能にする。

## 仕組みの変遷

### 従来の OCR（ルールベース・古典的 ML）

文字の形状をテンプレートと照合したり、手作りの特徴量で分類する。印刷フォントには強いが、手書き・複雑なレイアウト・ノイズに弱い。

```
画像 → 前処理（2値化・傾き補正）→ 文字領域の検出 → 文字分類（テンプレートマッチング）→ テキスト
```

### 深層学習ベースの OCR

CNN で文字の視覚特徴を抽出し、RNN/Transformer で文脈を考慮して認識する。手書き・低品質スキャン・複雑なレイアウトにも対応できる。

```
画像 → CNN（特徴抽出）→ Transformer（文脈推論）→ テキスト
```

現在の主要な OCR エンジンはこのアーキテクチャが主流。

### マルチモーダル LLM による OCR

画像をそのままLLMに渡して文字を読み取る。単なるテキスト抽出を超えて、**レイアウトの意味・表の構造・文脈の理解**まで含めた処理ができる。

```
請求書の画像 → LLM → 「金額: 50,000円、支払期日: 2024/03/31、品目: ○○」
```

## 用途別の使い分け

| 用途 | 向く手法 |
|---|---|
| 大量の定型帳票（フォーマットが固定） | 専用 OCR エンジン（高速・コスト低） |
| 手書き書類・非定型文書 | 深層学習 OCR |
| 表・レイアウトを含む複雑な文書 | マルチモーダル LLM |
| テキスト抽出後の意味理解・データ整形 | OCR + LLM の組み合わせ |

## OCR の精度を上げるコツ

### 前処理

```python
# OpenCV での前処理例
import cv2

img = cv2.imread('document.jpg', cv2.IMREAD_GRAYSCALE)

# コントラスト強調
img = cv2.equalizeHist(img)

# 2値化（大津の手法）
_, img = cv2.threshold(img, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

# ノイズ除去
img = cv2.medianBlur(img, 3)
```

- **解像度**: 300 DPI 以上が推奨。低解像度は精度が落ちる
- **傾き補正**: 数度の傾きでも精度が大きく変わる
- **2値化**: グレースケール→白黒に変換してコントラストを上げる

### LLM との組み合わせ

OCR でテキスト抽出した後、LLM で構造化・補正・理解をさせる構成が実用的。

```
画像 → OCR → 生テキスト（精度 80〜90%）→ LLM → 構造化・誤字補正・データ抽出
```

## マルチモーダル LLM との使い分け

| 観点 | 専用 OCR | マルチモーダル LLM |
|---|---|---|
| 速度 | 速い | 遅い（推論コストが高い） |
| コスト | 低い | 高い |
| 精度（定型） | 高い | 同等〜やや劣る |
| 複雑な文書 | 苦手 | 強い |
| レイアウト理解 | 限定的 | 強い（表・図の意味を理解） |
| 後処理の必要性 | 高い（生テキストの整形が必要） | 低い（整形済みで返せる） |

大量処理・コスト重視なら専用 OCR、複雑な文書や後処理も含めて任せたいなら LLM という使い分けが現実的。
