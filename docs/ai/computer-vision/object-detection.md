# 物体検出（Object Detection）

## なぜ存在するか

画像分類は「この画像は何か」を答えるが、現実の問題では「画像の中のどこに何があるか」を知る必要がある。
物体検出は画像内の物体の**位置（バウンディングボックス）とクラス（種類）を同時に特定する**タスク。

```
画像分類:  画像全体 → 「犬」
物体検出:  画像全体 → 「犬（左上 x:120 y:80 w:200 h:180）」
                       「人（右 x:400 y:50 w:150 h:300）」
```

## 仕組みの変遷

### 従来の手法（スライディングウィンドウ）

画像上でウィンドウをずらしながら全領域を分類する。精度は出るが計算コストが極めて高い。

### R-CNN 系（2段階検出）

1. 候補領域（Region Proposal）を先に抽出
2. 各候補領域を分類

精度は高いが、2段階のためリアルタイム処理には向かない。

### YOLO（You Only Look Once）

画像を一度だけ処理して検出と分類を同時に行う1段階検出。

```
画像 → グリッド分割 → 各セルがバウンディングボックス + クラス確率を予測 → NMS で重複除去 → 結果
```

「1度見るだけ（You Only Look Once）」という名前の通り、従来より格段に高速。リアルタイム処理を実現した。

## YOLO の特徴

### グリッドベースの予測

画像をグリッドに分割し、各セルが「物体の中心がこのセル内にあるか」を予測する。1つのセルが複数の物体候補（アンカー）を予測する。

```
13×13 グリッド → 各セルが 3 アンカー × (4座標 + 信頼度 + クラス数) を出力
```

### NMS（Non-Maximum Suppression）

同一物体に対して複数の重複したボックスが予測されるため、最も信頼度の高いものだけを残す後処理。

### バージョンの進化

| バージョン | 特徴 |
|---|---|
| YOLOv1〜v3 | 基本アーキテクチャの確立 |
| YOLOv5 | PyTorch実装。使いやすさと精度のバランス |
| YOLOv8 | Ultralytics製。セグメンテーション・姿勢推定にも対応 |
| YOLOv11 | 最新版。軽量化と高精度を両立 |

## 関連タスク

物体検出を拡張した派生タスク。

| タスク | 内容 |
|---|---|
| 物体検出 | バウンディングボックス + クラス |
| セグメンテーション | ピクセル単位で物体領域を塗り分ける |
| 姿勢推定（Pose Estimation） | 人体の関節点を検出する |
| 追跡（Tracking） | 動画内で物体を継続的に追う |

## 精度指標

### mAP（mean Average Precision）

各クラスの AP（適合率-再現率曲線下の面積）を平均した指標。高いほど良い。

- **IoU（Intersection over Union）**: 予測ボックスと正解ボックスの重なり率。0.5 以上を正解とすることが多い

```
IoU = 重なり面積 / 合計面積
```

## いつ使うか

- 画像・動画内に複数の物体が存在し、それぞれの位置も必要なとき
- リアルタイムでの検出が必要なとき（監視カメラ・自動運転・ロボット）
- 商品の欠陥検査・工場での品質管理
- 医療画像での病変領域の特定

**画像全体のラベルだけでよければ画像分類で十分**。位置情報が不要なら物体検出のコストは高すぎる。
