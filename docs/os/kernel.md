# カーネル（Kernel）

OSの中核となるソフトウェア。ハードウェアリソース（CPU・メモリ・ディスク・ネットワーク）を管理し、アプリケーションがハードウェアを直接触らなくて済むよう抽象化する。

## なぜ存在するか

複数のプログラムが同時にハードウェアに直接アクセスすると、衝突・クラッシュ・セキュリティ侵害が起きる。カーネルはハードウェアへの唯一の窓口となり、リソースの公平な配分・プロセス間の隔離・アクセス制御を担う。

## カーネル空間とユーザー空間

```
┌──────────────────────────────────┐
│  ユーザー空間（User Space）        │
│  アプリケーション・シェル・ライブラリ │
│  → ハードウェアに直接触れない       │
├──────────────────────────────────┤
│  システムコール境界（syscall）      │  ← 唯一の入口
├──────────────────────────────────┤
│  カーネル空間（Kernel Space）      │
│  プロセス管理・メモリ管理            │
│  ファイルシステム・ネットワークスタック │
│  デバイスドライバ                  │
├──────────────────────────────────┤
│  ハードウェア                      │
└──────────────────────────────────┘
```

ユーザー空間のプロセスはシステムコール（`read`・`write`・`fork`・`exec` など）を呼び出すことでカーネルに処理を依頼する。カーネル空間への直接アクセスはCPUの保護機構（リングレベル）によって禁止されている。

## カーネルの主な役割

| 役割 | 内容 |
|------|------|
| プロセス管理 | プロセスの生成・スケジューリング・終了・プロセス間通信 |
| メモリ管理 | 仮想メモリ・ページング・各プロセスへのメモリ割り当て |
| ファイルシステム | ディスクI/Oの抽象化・VFSによる統一インターフェース |
| ネットワーク | TCPスタック・ソケット・パケット送受信 |
| デバイス管理 | デバイスドライバを通じてハードウェアを操作 |
| セキュリティ | DAC・MAC・Capabilities・namespaceによる隔離 |

## カーネルの種類

### モノリシックカーネル

ファイルシステム・デバイスドライバ・ネットワークスタックなどをすべてカーネル空間で実行する。パフォーマンスが高い反面、1箇所のバグがシステム全体に影響しうる。**Linux・UNIXはこの方式**。

### マイクロカーネル

最小限の機能（プロセス間通信・メモリ管理など）だけをカーネルに置き、他はユーザー空間のサービスとして動かす。安定性・移植性が高いが通信オーバーヘッドがある。QNX・Mach（macOSの基盤）など。

## ペネトレーションテストでの意味

カーネルの脆弱性を悪用することで、一般ユーザー権限からroot権限への**権限昇格**が可能になる。カーネルはすべてのプロセスより上位で動作するため、カーネル自体が侵害されると完全な制御を奪われる。
