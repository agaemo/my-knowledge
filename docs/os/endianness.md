# エンディアン（Endianness）

複数バイトのデータをメモリ上に格納するときのバイト順序。リトルエンディアンとビッグエンディアンの2種類がある。

## なぜ存在するか

CPUがメモリから複数バイトを読み書きするとき、どのバイトを先頭アドレスに置くかはアーキテクチャの設計判断による。歴史的に異なるアーキテクチャが独立して発展した結果、2種類の慣習が定着した。

## リトルエンディアンとビッグエンディアン

4バイトの値 `0x12345678` を格納する例:

```
アドレス:    0x00  0x01  0x02  0x03

リトルエンディアン（x86・x86-64・ARMデフォルト）:
            0x78  0x56  0x34  0x12   ← 下位バイトが先頭（低いアドレス）

ビッグエンディアン（ネットワークバイトオーダー・一部RISC）:
            0x12  0x34  0x56  0x78   ← 上位バイトが先頭（低いアドレス）
```

- **リトルエンディアン**: Intel/AMD（x86・x86-64）・ARM（デフォルト）など現代の主要CPU
- **ビッグエンディアン**: ネットワークプロトコル（TCP/IP）・SPARC・一部の組み込みCPU

## いつ意識するか

### バイナリ解析・フォレンジクス

実行ファイル（ELF・PE）やファイルフォーマットを手動で読む場合、アーキテクチャのエンディアンを把握していないと数値を誤読する。

### ネットワークプロトコル

TCP/IPはビッグエンディアン（ネットワークバイトオーダー）を採用している。x86ホストと通信するコードでは `htons()` / `ntohs()` 等でバイト順を変換する。

### エクスプロイト開発

バッファオーバーフロー等でアドレスをペイロードに埋め込むとき、ターゲットのエンディアンに合わせてバイト順を逆にして渡す必要がある。

```
# x86-64（リトルエンディアン）でアドレス 0xdeadbeef を埋め込む例
\xef\xbe\xad\xde  ← 下位バイトから順に並べる
```

### CTF・リバースエンジニアリング

メモリダンプやパケットキャプチャから値を読み取るとき、エンディアンの変換ミスがそのまま解析の失敗につながる。
