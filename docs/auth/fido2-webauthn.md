# FIDO2 / WebAuthn / Passkeys

パスワードに依存しない認証の標準仕様。「パスワードレス認証」の技術的な基盤。

## 3つの用語の関係

混同されやすいが、それぞれ役割が異なる。

```
FIDO2
  ├── WebAuthn  — ブラウザ・アプリが認証器と通信するための API（W3C 標準）
  └── CTAP      — 認証器（YubiKey・スマートフォン）との通信プロトコル

Passkeys — FIDO2 を一般ユーザー向けに使いやすくしたもの（Apple・Google・Microsoft が推進）
```

**FIDO2** が仕様全体、**WebAuthn** がその Web 向け実装、**Passkeys** がユーザー体験の名称、という関係。

## なぜ生まれたか

パスワード認証には構造的な問題がある。

- **フィッシング：** 偽サイトにパスワードを入力させて盗む
- **パスワード使い回し：** 1か所の漏洩が他サービスへの不正ログインにつながる
- **サーバー側の漏洩：** DB が流出するとハッシュから解析される

FIDO2 はこれらをまとめて解決するために、**パスワードを認証の仕組みから取り除く**設計で作られた。

## 仕組み

公開鍵暗号を使う。ユーザーのデバイスで鍵ペアを生成し、秘密鍵はデバイス外に出ない。

```
登録:
  1. デバイスが公開鍵・秘密鍵のペアを生成
  2. 公開鍵だけをサーバーに登録する
  3. 秘密鍵はデバイスの安全領域（Secure Enclave・TPM）に保存

認証:
  1. サーバーがランダムな「チャレンジ」を送る
  2. デバイスが秘密鍵でチャレンジに署名する（生体認証・PIN で本人確認）
  3. サーバーが公開鍵で署名を検証する
```

パスワードがネットワークを流れないため、フィッシングで盗む手段がない。

## フィッシング耐性

FIDO2 最大の特徴。鍵ペアは **登録時のドメインと紐づいている**。

```
example.com で登録した鍵 → evil-example.com では使えない
```

偽サイトに誘導しても、そのサイト向けの認証情報が存在しないためログインできない。パスワード認証では入力してしまえば盗まれるが、FIDO2 では構造的に防がれる。

## Passkeys とは

FIDO2 の普及を妨げていた「デバイスをなくしたら詰む」問題を解決するために登場した。

**従来の FIDO2（ハードウェアキー）：**
- YubiKey などの物理デバイスに鍵が閉じ込められる
- デバイスを紛失すると復旧が困難

**Passkeys：**
- 秘密鍵をクラウド（iCloud キーチェーン・Google パスワードマネージャー）で同期
- 同じアカウントのデバイス間でシームレスに使える
- iPhone で登録 → Mac・iPad でもそのまま使える

Apple・Google・Microsoft が足並みを揃えて推進しており、2022年以降に急速に普及した。

## パスワードレスと MFA の違い

| | パスワード + MFA | FIDO2 / Passkeys |
|---|---|---|
| パスワード | 必要 | 不要 |
| フィッシング耐性 | 低い（SMSコードは盗める） | 高い（構造的に防ぐ） |
| UX | 手順が多い | 生体認証1回で完了 |
| 普及状況 | 現在の主流 | 急速に普及中 |

SMS や TOTP（Google Authenticator）による MFA はフィッシングで突破できるケースがある。FIDO2 はこれを構造的に解決する。

## 認証器の種類

**プラットフォーム認証器：** デバイス内蔵。Touch ID・Face ID・Windows Hello など。Passkeys はこれを使う。

**ローミング認証器：** 外付け。YubiKey・FIDO セキュリティキーなど。USB・NFC で接続。エンタープライズや高セキュリティ要件の環境で使われる。

## いつ採用するか

```
採用を検討する場面:
  ✓ フィッシング被害のリスクが高いサービス（金融・企業システム）
  ✓ パスワード管理の運用コストを下げたい
  ✓ 「Google でログイン」以外のパスワードレス体験を提供したい

現状の注意点:
  - 古いブラウザ・OS では未対応のケースがある
  - パスワードのフォールバックをどう設計するか検討が必要
  - エンタープライズ向けはまだ SAML / OIDC との組み合わせが多い
```
